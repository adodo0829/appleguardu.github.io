Vue学习路线

发表于 2019-03-09 | 分类于 Vue

写在前面:
当我们买了一本新书, 我们最先开始看的一般都不是内容, 而是去把目录过一遍,看一下有多少章节呀; 同理,学习一项新技术时,我觉得一般要先去了解一下这个技术整体架构内容, 需要有一种大局观, 然后是去了解了各个章节直接的关联关系后,再去逐步深入学习…
下面整一下 我的Vue学习路线和计划, 详细内容会在学习之后针对每个章节做一些总结

基础语法学习

首先, 过一下官方文档,去看看Vue有哪些内容组成,然后列一个由浅入深的学习计划

  • 基本的指令

    像 v-指令名,这些是首先去需要掌握的, 条件,循环,属性绑定, 事件绑定, 修饰符等

  • Vue实例的创建

    记住Vue实例的创建流程
Vue实例的一些重要且常用的属性和方法
    data, methods, computed, watch等...深入理解 + 实操
实例属性: 以$开头访问的属性

  • Vue实例的生命周期

    记住八大生命周期钩子(回调函数): 干了什么事..
vue实例在各个阶段的回调函数: 总结一下在这期间可以做什么事, 比如
    beforecreate: 可以在这加个loading事件
    created：做一些初始化，可以访问data
    mounted：在这发起后端请求，拿回数据，配合路由钩子做一些事情
    updated: 数据更新后做一些事
    beforeDestory： 你确认删除XX吗？ 
    destoryed ：当前组件已被删除，清空相关内容

组件的学习(重点)

Vue组件作为独立的可重用ui元素, 需要掌握好。要知道页面的实现: 是先进行拆分成多个组件, 然后再拼凑组件的一个过程. 考虑到后期的可扩展和移植性, 组件一定要独立

  • 组件的基本使用

    比如如何创建,注册,添加方法,属性等...

  • 组件通信传值(重要)

    如何通过 prop 和 event 在它们之间发生交互。
    • 数据单向流
    • 父子组件
    • 兄弟组件
    • 跨级组件

单页面应用程序(SPA)

单页面应用程序（SPA）架构通过单个网页实现传统多页面网站一样的功能，而且不会在每次用户触发导航时重新加载和重建页面;

学会Vue Router

Vue Router 是一个用于构建 SPA 的工具:
将“页面”构建为 Vue 组件之后，就可以使用 Vue Router 将每个“页面”映射到一个唯一的路径

router-link-to
router-view
路由设置, 动态路由, 二级路由(嵌套)

状态管理

随着应用程序变得越来越大，项目变得越来越复杂，SPA 页面中会有很多组件，管理全局状态变得异常困难，而且随着 prop 和 event 监听器的增加，组件变得越来越臃肿

学会Vuex

基于Flux思想: 数据单向流动思想，它可以将数据保存在可预测且稳定的中央存储中;

脚手架Vue-cli

能够快速搭建开发环境,设置一些配置

Webpack

模块打包器
代码构建工具: 能够实现按需求定制 来配置文件
