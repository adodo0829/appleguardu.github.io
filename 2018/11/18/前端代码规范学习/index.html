前端代码规范

发表于 2018-11-18 | 分类于 前端学习

一个良好的编码习惯可以提升我们的开发效率,本篇是关于编程日常中的一些代码规范,习惯的总结…

代码规范

文件命名规范

1.项目命名:
    全部采用小写方式， 以下划线分隔; 如 my_project_name
2.目录命名:
    同上; 但有复数结构时，要采用复数命名法; 如images
3.html,css,js文件命名:
    同上

HTML中的一些规范

1.结构语法
    嵌套节点缩进4个空格; 属性名使用小写,中线分隔符,双引号;
2.文件引入
    link, script标签引入 对应的css, js文件; 不需要额外申明type类型
3.节点的属性编写遵守的顺序
    class是一般为高可复用组件设计的...id具有唯一性, 尽量少使用
    常用节点元素:类class, id, name, 自定义属性data-xxxx, src/href/value等等...
    表单元素: 同上 + type, title/alt/placeholder + 布尔值参数: checked,disabled, + required,readonly等等...
4.标签的使用频次
    a.尽量避免多余的父节点标签
    b.避免使用js生成标签, 不便于查找
    c.尽量用最少的标签和复杂度来实现 所需要展现的结构

CSS中的一些规范

1.书写格式
    每个属性名: 缩进4格,属性值空一格,多个选择器逗号分隔换行;
2.空行:
    每个选择器样式之间空一行; 把功能相似的属性名写在一起,分区, 每一类空一行
3.引号
    url路径引用,属性值content="",属性选择器都使用双引号
4.命名
    类名使用小写字母，以中划线分隔; id采用驼峰式命名;
    scss中的变量、函数、混合、placeholder采用驼峰式命名
5.属性书写的顺序:
    a.先写确定元素显示的一类属性: 如display","visibility","float","overflow",
    b.再写决定元素布局方式的一类属性: flex,table-layout...
    c.定位的一些属性
    d.盒子模型的一些属性: 内外边距, 边框...
    e.盒子内容的一些样式属性: 字体,颜色,位置...
    f.盒子背景等属性....
    g.CSS3中的一些动画属性....
6.颜色16进制,尽量简写; #000
7.属性简写
    属性简写需要你非常清楚属性值的正确顺序，而且在大多数情况下并不需要设置属性简写中包含的所有值，
    所以建议尽量分开声明会更加清晰；特别是c3中的一些动画属性;

JS中的一些规范

1.缩进: 4格
2.分号: 变量声明;表达式;return;throw;break;continue;do-while;等
3.空格: 很多地方都要注意,比如你觉得看的不舒服的地方就要加...这里就不写了
4.空行: 对象中的属性对应的是函数
5.注释: 单行加一个空格,对齐代码块; 多行最少三个* 
    /*
     * xxxxxx
     */
6.文档注释,函数注释
    /**
     * @func
     * @desc 一个带参数的函数
     * @param {string} a - 参数a
     * @param {number} b=1 - 参数b默认值为1
     * @param {string} c=1 - 参数c有两种支持的取
     */
7.最外层使用单引号
8.变量命名:
    标准变量采用驼峰式命名（除了对象的属性外，主要是考虑到cgi返回的数据）
    常量全大写，用下划线连接 MAX_COUNT
    构造函数，大写第一个字母
    jquery对象必须以'$'开头命名 $body = $('body')
9.变量声明:
    一个函数作用域中所有的变量声明尽量提到函数首部
10.数组、对象
    对象属性名不需要加引号；
    对象以缩进的形式书写，不要写在一行；
    数组、对象最后不要有逗号。

11.null关键字的使用场景: 主要用于对象
    初始化一个将来可能被赋值为对象的变量
    与已经初始化的变量做比较
    作为一个参数为对象的函数的调用传参
    作为一个返回对象的函数的返回值
12.undefined
    永远不要直接使用undefined进行变量判断；
    使用typeof和字符串'undefined'对变量进行判断。

13.对上下文this的引用只能使用'_this', 'that', 'self'其中一个来命名；
