css基础性总结

发表于 2018-10-28 | 分类于 前端学习

关于css阶段性的学习总结…

css选择器

css选择器是用于对网页中的元素进行样式的控制，比如文字，背景，图片等等…

  • 选择器的分类 

    用的比较多的是标签，类，id，像*，！important这些比较少用

  • 复合选择器 

    后代选择器，交集选择器(li.name,提高元素控制精确度，少用)，并集选择器(,)，伪类选择器(a:hover/li:nth-child等...),子代选择器(li>a)

  • 注意: 在使用选择器作用某一个元素时，选择器不要太冗长，一般1-3个

    对用属性相同的元素类名可以复用，还要注意复合选择器的权重和优先级。

css的三大特性

继承，层叠，优先级

  • 层叠性 

    多个选择器(权重相同情况下)作用到同一标签，发生属性冲突，后来者居上

  • 继承性

    存在嵌套关系的元素，子元素会继承父元素的部分属性
特殊性（a标签的颜色，h标签的大小）

  • 优先级 

    继承 < 通配符 < 标签选择器 < 类选择器 < id选择器 < 行内式 < ！important

css中的元素的显示模式

行内元素: 一行显示多个，不能设置宽高；如标签span a ins strong del…..

块级元素: 单独占据一行，可以设置宽高；如标签div h p ul li…..

行内块元素: 具有以上两者的特点，一行显示多个，可以设置宽高；如标签img input……

模式之间的转化: 可以通过display: inline block inline-block进行模式转化

css中常用的样式属性

  • 字体 

    大小，字体库，颜色，style，行高lh（文字居中的关键，也可调节多行文字的间距）

  • 背景 

    bgc(可以写颜色名,rgb,rgba,注意transparent的使用)
尽量使用连写方式，bg：url 是否平铺 位置(可以是xy坐标，可以是方位) 颜色
注意：bga：fixed；下拉滚动条，背景不动

  • 插入图片和背景图片的区别

    1. 插入图片 我们用的最多 比如产品展示类  移动位置只能靠盒模型 padding margin
2. 背景图片我们一般用于小图标背景 或者 超大背景图片  背景图片 只能通过  background-position
3. 图片去除间隙方法：转为块级； vertical-align

盒子模型

盒子的组成：内容，边框border，内边距padding（内容与边框的距离），外边距margin（盒子之间的距离）
margin，padding，border都有四边，属性连写顺序顺时针方向top right bottom left,页面布局中经常用到，学会合理的使用！！！

对于盒子的理解：
网页中的标签元素其实都可以看作一个盒子，我们可以把盒子看作是一个存放内容（图片，文本，链接…）的容器，这个容器有它的高宽，内容，还有和内容的间距，以及容器之间的间距。最常用的盒子：div标签

盒子的真实宽高 = content-width/height + padding + border 

盒子水平居中,这个在项目中经常用到，
margin:0 auto 
1.块级block  2.盒子有宽度width

盒子常用属性和注意点： 

1.常用属性：
宽高，边框，内边距，外边距，圆角，背景，overflow等等....
2.注意点：
2.1 padding和margin对行内元素不起作用
2.2 避免垂直塌陷(上下盒子排布会取较大的margin)；包含塌陷(嵌套关系的盒子，子盒子设置mt会带偏父盒子，给父元素添加overflow或者mt)
2.3 元素隐藏display:none，位置不保留
2.4 文字在盒子中溢出显示
    white-space:nowrap
    text-overflow: ellipsis
    overflow:hidden
2.5 如何盒子水平方向内容经常变化，宽度不写死，用内边距撑开
2.6 盒子的高度一般情况下也不宜写死，由内容来撑开

关于浮动

浮动：给元素添加float属性，会使元素脱离标准流，浮动起来。。。

浮动的特点

1-浮动的元素，实现了模式的转换（拥有了行内块的特点）
2-浮动的元素，顺序与结构代码有关，依次排列。
3-浮动的元素，脱离了标准流，层级比标准流高，会浮在标准元素之上,标准 流会去占据浮动元素的位置。
4-浮动元素是顶部对齐。
5-浮动的父元素会把子元素带跑
6-浮动的子元素依旧会在父元素范围内
7-如果浮动的元素前面有标准流，那么浮动元素只能在标准流之后浮动

浮动的影响和解决
在项目中，我们使用嵌套盒子时，一般不会给父盒子设高度，当父盒子是标准流时，若子盒子又浮动起来，脱离了标准流，父盒子就会丧失高度，其他标准流的元素就会来占据他的位置，造成布局混乱。
解决措施：给父盒子清除浮动：高度法，单双伪元素法，添加overflow属性
这里写一个单伪元素法,5个属性！！！
.clearfix:after {
content：”.”;
display: block;
clear: both;
visibility: hidden;
height: 0;
}

关于定位

  • 相对定位 

    不会转换显示模式，没有脱离标准流，还会保留原来的位置，但是显示的层级会变高，一般用于配合绝对定位。

  • 固定定位

    实现了模式转化，具有行内块的特点；脱离了标准流，不会占据标准流的位置；位移参照物为body。

  • 绝对定位 

    实现了模式转化，具有行内块的特点；脱离了标准流，不会占据标准流的位置；  
位移参照物为最近的具有定位属性的父盒子。

  • 显示层级 

    定位 >  浮动  >  标准流

关于css+div布局

静态页面布局的一般步骤：

  • 1.将网页先拆分纵向分布的大盒子
  • 2.再对大盒子进行逐级拆分
  • 3.选择合适的方式(标准流，浮动，定位)，先分析目标盒子的特征，比如是一行显示多个，还是多行显示，或者是混搭。

常用技巧：

  1. 类似头部导航栏一行显示多个，可以采用ul>li标签+浮动来做
  2. 个别独立的小模块可以用一个盒子写好，直接定位过去。
  3. 图文组合在一个盒子中的，文字换行存在间距的用行高撑开比较好
  4. logo的写法，a+背景图片
  5. 底部footer栏，dl+dt>span + dd>a,然后浮动
  6. 底部about栏，直接用标准流写比较好 a+|的形式
  7. 精灵图 修改背景图的bgp即可
  8. 版心布局
  9. 滑动门的原理，不定宽度，父子元素使用同一个背景图片，距离padding撑开。

移动端的自适应布局

使用根元素字体大小 rem + 视口大小vm 来实现网页结构的自适应; 随视口的变化而变化…

思路:
// 根元素字体大小设置: 设置css样式中根元素字体大小 1rem = 100px; // 一般我们会默认为100px,方便计算
// 这样网页中的其他元素以根元素字体为参照物(100px)来写
// 比如图片宽60px, 就是0.6rem; 

// 但是我们要实现自适应: 
// 推理过程: 即实现元素的大小随着视口宽度vm变化而变化 => 根元素字体大小随vm变化而变化(线性关系)
// 所以我们要将根元素字体大小转为视口宽度:  相当于我要把设计稿按比例投放到设备上去

// 公式:
// 根元素字体实际大小(可以理解为假设设计稿根元素100px)/设计稿大小 = 根元素视口大小/设备视口大小(100vm)  
// (当然这是在设计稿尺寸:设备尺寸 = 1:1的时候)

// 如果实际上设计稿:设备尺寸 = 2:1; 意味着我需要将设计稿缩小到1/2才能投放到设备上完全展现; 
// 但是我的根元素实际大小和视口大小为定值(100px; 100vm);
// 所以只有将设计稿尺寸缩小一倍; 那个根元素的视口大小就要 乘以2...
// 如果是3倍; 根元素视口大小则 x3...

less使用总结

1.将重复使用的样式放在变量中 @变量: 常用属性1
2.&符号: 类似js中的this, 表示当前选择器下那个父级对象
3.混入写法, 可以直接在一个类中写另一个类名, 表示继承其所有属性;
4.嵌套写法: 一般在写less文件时, 遵循嵌套规则, 先写父级元素, 也不要着急去写样式, 
    再把一层层的子元素先列出来, 再按照设计图去一次添加属性,这样代码思维会比较流畅...

sass使用

1.定义变量: $变量名: 样式值;
2.嵌套: 同上
3.混入写法: @mixin  混入名(变量...) {设置属性和值}; 引用混入: @include 变量
4.导入: @import '路径'; 同上
